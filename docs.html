Getting Started

System Requirements

  • Modern web browser for the research dashboard
  • SSH client for cluster access
  • Python 3.10+ for SDK and local development
  • NVIDIA drivers (if running local GPU workloads)

Access & Authentication

All platform access is provisioned through your organisation's admin. Once onboarded, you'll receive:

  • API credentials (RICCHE_API_KEY)
  • SSH keys for GPU cluster access
  • Dashboard login via SSO (SAML 2.0 / OIDC)

Quick Start

Install the Ricche SDK and authenticate:

pip install ricche-sdk

ricche auth login

Submit your first experiment:

ricche experiment submit --config experiment.yaml

This will queue your job on the GPU cluster, stream logs, and store results in the experiment registry.

Platform Overview

Ricche provides five integrated systems that work together as a unified research platform:

  • Data Infrastructure — Automated pipelines for market data ingestion, normalisation, and quality validation.
  • ML Research Environment — GPU-accelerated training on NVIDIA H200 clusters with PyTorch, JAX, and custom CUDA kernels.
  • Simulation Framework — Backtest across thousands of market scenarios using GPU-parallelised Monte Carlo methods.
  • Validation & Governance — Experiment tracking, model versioning, and full audit trail for regulatory compliance.
  • Monitoring & Observability — Real-time dashboards for pipeline health, GPU utilisation, and experiment drift detection.

API Reference

Full API documentation is available to authenticated users. Key endpoints:

  • POST /v1/experiments — Submit a new experiment
  • GET /v1/experiments/{id} — Retrieve experiment status and results
  • GET /v1/data/feeds — List available market data feeds
  • POST /v1/backtest — Run a GPU-accelerated backtest
  • GET /v1/signals — Query the alpha signal library

Detailed request/response schemas and authentication guides are provided upon platform access. Request access to receive full API documentation.
